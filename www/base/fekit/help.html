2.关于个人模块页使用说明

所有的模块代码都应该包含在一个 类名为 cobra_system_moudle_list 的 ul 中，每个模块使用 类名为 cobra_system_moudle_item 的 li 包裹；同时，这个 li 还可以有另外两个类名：cobra_system_moudle_ph10 和 cobra_system_moudle_demo.

cobra_system_moudle_ph10 : 为模块容器在水平方向左右两边各添加 10px 的空隙，翻遍对模块进行查看，若所编写的模块的边界需要和浏览器边界紧挨，只需去掉该类名即可。

cobra_system_moudle_demo : 为模块添加视觉样式的类名。即：为了能方便的看到我们所编写的模块的效果，很多时候需要为模块添加一些诸如 背景、边框等样式，但这些样式可能并不是模块本身所需要的，只是为了更好的观察效果而临时添加的。例如，我们编写了一个框架模块，有 main 和 side 两列，这时，我们就可以通过该类名分别为 main 和 side 添加不同的背景色,即可非常直观的看到 main 和 side 的位置：

.cobra_system_moudle_demo .main{background:#333;}
.cobra_system_moudle_demo .side{background:#CCC;}

所有和模块本身无关的代码都应该使用 cobra_system_moudle_demo 进行控制，以免造成不必要的代码冗余。

所有此类代码都位于个人 css 或 less 文件中:

/************************************* demo start ******************************/

/************************************* demo end ********************************/

注释块内，方便统一管理。
